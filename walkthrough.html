OpenWRT Monitor

A real-time, widget-based monitoring dashboard for your OpenWRT router. Track CPU, memory, traffic, temperature, and connected devices — all from a beautiful, installable PWA.

Get Started Learn More
● Overview

What does it do?

OpenWRT Monitor polls your router's LuCI JSON-RPC API on a configurable interval, stores historical data in SQLite for up to 30 days, and serves a drag-and-drop dashboard in your browser.

⚡

Real-Time Metrics

CPU, RAM, load averages, temperature, and network traffic updated every 10 seconds.

📊

Historical Data

SQLite stores up to 30 days of snapshots with automatic daily purge at 3 AM.

📱

PWA Installable

Service Worker caching + web manifest — install it to your home screen on any device.

🧩

Drag & Drop Widgets

10 built-in widgets powered by Gridstack. Rearrange, add, remove, and persist layouts.

🌗

Dark & Light Themes

Fully themed UI with smooth transitions. Your preference is saved in localStorage.

📈

Chart.js Graphs

Interactive time-series charts for CPU, memory, load, traffic, and temperature history.

● Prerequisites

Before you begin

Make sure these are available on the machine that will host the dashboard.

⬢

Node.js ≥ 18

Required for the Express backend and better-sqlite3 native module. Verify with node -v.

📦

npm ≥ 9

Ships with Node.js. Used to install dependencies. Verify with npm -v.

🖧

OpenWRT Router

Running a recent OpenWRT release with LuCI installed and the JSON-RPC API enabled.

🔌

Network Access

The host machine must be able to reach the router's IP over HTTP (default port 80).

● Router Setup

Enable LuCI JSON-RPC

The dashboard communicates with your router through LuCI's JSON-RPC interface. This is usually bundled with LuCI but may need to be explicitly installed.

1

SSH into your router

ssh root@192.168.1.1
2

Install the RPC packages

Install the LuCI JSON-RPC modules if they're not already present. OpenWRT 24.x and newer use apk (Alpine Package Keeper) instead of the legacy opkg:

# OpenWRT 24.x+ (apk)
apk update
apk add luci-mod-rpc luci-compat
ℹ️

Legacy OpenWRT (pre-24.x): If you're on an older release that still uses opkg, run opkg update && opkg install luci-mod-rpc luci-lib-ipkg luci-compat instead.

3

Verify the RPC endpoint

From your host machine, test that the auth endpoint responds:

curl -s -X POST http://192.168.1.1/cgi-bin/luci/rpc/auth \
  -H "Content-Type: application/json" \
  -d '{"id":1,"method":"login","params":["root","your_password"]}'

You should receive a JSON response with a session token in "result". If you get "result": null, double-check your password and that the RPC packages are installed.

⚠️

Security note: The RPC interface transmits credentials in plain text over HTTP. If your router is accessible beyond your LAN, consider setting up HTTPS on LuCI or restricting access via firewall rules.

● Installation

Install the dashboard

Clone the repository and install dependencies on your host machine (macOS, Linux, or Windows with WSL).

1

Clone the repository

git clone <your-repo-url> openwrt-monitor
cd openwrt-monitor
2

Install dependencies

This installs Express, better-sqlite3, Vite, and all other packages:

npm install
ℹ️

better-sqlite3 compiles a native C++ addon. You may need build tools installed (xcode-select --install on macOS, or build-essential on Ubuntu).

● Configuration

Configure your environment

All runtime configuration is managed through a .env file in the project root.

1

Create the .env file

cp .env.example .env
2

Edit the values

Open .env in your editor and fill in the real values:

# OpenWRT Router Configuration
ROUTER_IP=192.168.1.1
ROUTER_USERNAME=root
ROUTER_PASSWORD=your_actual_password

# Application Settings
POLL_INTERVAL_SECONDS=10
PORT=3000

Environment Variables Reference

🔒

Never commit your .env file. It's already in .gitignore. Keep your router password safe.

🛡️

Secure Local Storage: This dashboard automatically saves your layout and theme to your browser's localStorage. To protect against tampering and XSS attacks, the layout configuration is strictly validated against a hardcoded widget schema every time you load the page.

● Running

Start the application

There are two modes: development (hot-reload) and production (optimized build).

▶

Development mode

Starts both the Express API server and the Vite dev server with hot module replacement:

npm run dev

This runs concurrently to start:

  • Backend → http://localhost:3000 (Express API via nodemon)
  • Frontend → http://localhost:5173 (Vite dev server)

Open http://localhost:5173 in your browser. Vite automatically proxies /api/* requests to Express.

🚀

Production mode

Build the frontend and serve everything from Express:

# 1. Build the frontend
npm run build

# 2. Start the production server
npm start

The full app (API + static files) is served at http://localhost:3000.

🧪

Test router connection

Run the built-in API test script to verify your router connection:

npm test
✅

On successful startup you'll see: [Server] Router connection successful! followed by [Poller] Started — polling every 10s. A "Loading data..." screen will appear initially while the system polls the first batch of metrics. Please wait a few seconds for the data to populate your dashboard.

● Troubleshooting

Common issues

!

"Unexpected token <" error

If you run into an "Unexpected token <" error when starting the server, it simply means your router's LuCI RPC packages aren't fully initialized — just make sure you've run apk add luci-mod-rpc luci-compat and restarted your router's web interface.

!

"Authentication failed" on startup

The ROUTER_PASSWORD in your .env is incorrect, or the LuCI RPC packages are not installed. SSH into the router and verify with the curl test from the Router Setup section.

!

Dashboard shows "Disconnected"

The backend can't reach the router. Check that ROUTER_IP is correct, the router is powered on, and there's no firewall blocking port 80 between your host and the router.

!

better-sqlite3 build failure

On macOS, run xcode-select --install first. On Linux, install build-essential python3. On Apple Silicon, make sure you're using an ARM-native Node.js build.

!

Temperature widget shows "N/A"

Not all OpenWRT devices expose thermal zones. If /sys/class/thermal/thermal_zone*/temp doesn't exist on your router, the temperature widget will remain empty.

!

Port 3000 already in use

Change the PORT value in .env to another port (e.g., 3001). If you change it, also update vite.config.js proxy target to match.
